# ExamVaultPlus Installation and Run Guide

This guide explains how to install and run the ExamVaultPlus JavaFX application on Windows.

## Prerequisites

1. **Java Development Kit (JDK)** - Version 17 or higher
   - Download from: https://adoptium.net/temurin/releases/
   - Or Oracle JDK: https://www.oracle.com/java/technologies/downloads/

2. **JavaFX SDK** - Version compatible with your JDK
   - Download from: https://gluonhq.com/products/javafx/
   - Choose the SDK for Windows

3. **Oracle JDBC Driver** (ojdbc8.jar)
   - Download from Oracle website or Maven Central
   - Place in `lib/` folder

4. **Oracle Database** - FreeSQL account at db.freesql.com
   - Create account and note your credentials
   - Schema: SRI_DIU_23_SCHEMA_9617Q

## Installation Steps

### Step 1: Download and Extract
1. Download the ExamVaultPlus project files
2. Extract to a folder (e.g., `C:\Projects\ExamVaultPlus`)
3. Ensure the folder structure matches:
   ```
   ExamVaultPlus/
   ├── src/
   ├── lib/
   │   └── ojdbc8.jar
   ├── data/
   └── run.bat
   ```

### Step 2: Configure Paths
1. Locate your JDK installation path (e.g., `C:\Program Files\Java\jdk-17`)
2. Locate your JavaFX SDK path (e.g., `C:\Program Files\Java\javafx-sdk-17`)
3. Open `run.bat` in a text editor
4. Update the paths at the top:

   ```batch
   set JDK=C:\Program Files\Java\jdk-17
   set JAVAFX=C:\Program Files\Java\javafx-sdk-17\lib
   ```

   **Important**: Replace with YOUR actual paths!

### Step 3: Configure Database
1. Open `src/utils/DatabaseConfig.java`
2. Update the database credentials:

   ```java
   public static final String URL = "jdbc:oracle:thin:@db.freesql.com:1521/23ai_34ui2";
   public static final String USER = "YOUR_SCHEMA_NAME";  // e.g., SRI_DIU_23_SCHEMA_9617Q
   public static final String PASSWORD = "YOUR_PASSWORD";
   ```

3. Run the SQL scripts to create tables (provided in project documentation)

### Step 4: Compile and Run
1. Double-click `run.bat` or run from command prompt:
   ```
   cd C:\Projects\ExamVaultPlus
   run.bat
   ```

2. The application should compile and launch automatically

## Troubleshooting

### Common Issues:

1. **"javac is not recognized"**
   - JDK bin folder not in PATH
   - Update JDK path in run.bat

2. **"JavaFX runtime components are missing"**
   - JavaFX path incorrect
   - Ensure JavaFX lib folder is correct

3. **"Oracle JDBC driver not found"**
   - ojdbc8.jar missing from lib/ folder
   - Download and place the JAR file

4. **Database connection fails**
   - Check credentials in DatabaseConfig.java
   - Verify FreeSQL account is active
   - Check network connectivity

5. **Compilation errors**
   - Ensure JDK version is 17+
   - Check all source files are present
   - Verify module paths in run.bat

### Path Configuration Examples:

**For JDK 17:**
```
set JDK=C:\Program Files\Eclipse Adoptium\jdk-17.0.8.101-hotspot
```

**For JavaFX 17:**
```
set JAVAFX=C:\Program Files\Java\javafx-sdk-17.0.2\lib
```

**For JDK 21:**
```
set JDK=C:\Program Files\Java\jdk-21
set JAVAFX=C:\Program Files\Java\javafx-sdk-21\lib
```

### Alternative Run Methods:

**Command Line Compilation:**
```batch
set PATH=%JDK%\bin;%PATH%
javac --module-path %JAVAFX% --add-modules javafx.controls,javafx.fxml -d out src\*.java src\**\*.java
java --module-path %JAVAFX% --add-modules javafx.controls,javafx.fxml -cp out;lib\ojdbc8.jar ExamVaultPlus
```

**Using IDE (VS Code/IntelliJ):**
1. Set JDK in IDE settings
2. Add JavaFX library to classpath
3. Add VM options: `--module-path "C:\path\to\javafx\lib" --add-modules javafx.controls,javafx.fxml`
4. Run MainApplication.java
